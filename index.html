  1. A: AngularJS applications are websites that use AngularJS to extend HTML elements with directives and bind data to HTML attributes with expressions.
  2. A: A controller is responsible for providing appliaction behavior that supports the markup.
  3. A: Ng-model binds input,select, and textarea information to a property on the scope using the controller.
  4. A: The double curly brackets holding the variable and value of the variable of the element.
  5. A: $scope is the binding part between the HTML and the JavasScript which is the controller. Using scope will bind what information is enterd to the value of the variable. Allowing the code to use that value and implement it into functions.
  6. A: Dependency injections define components with services(actionable commands) and values(information) as dependencies. Since they are not hard coded within the HTML, they are resuable and editable. This makes dependency injections more versitile forms of information.
  7. A: Two-way binding, or data-binding is the synchronization between the view and the model components. The view displays the information, and the model holds and updates the infomation.
  8. A: The controller as syntax is another way to clarify what scope is being called. The controller is named first then the scope or action is then named. This helps define what scope is being accessed.
  9. A: AnglarJS is known as MV* architecture. On the server side it is MVC, which is basically like a call in order, the sales person sends the order to manufacturer. Who, once completed, ships out the order. The order goes from controller (sales rep) to the manufactures (model) to the the shipping company(the view). Where as MVVC takes input from the view and sends to the view model who in turn sends that input information to the model who then sends it back to the view model to be displayed back in the view. This is considered more client based and has more UI/UX emphasis.
  10. A: The main differece between injected functions, factories and services in AngularJS is the value of the function returned. After using module.service one will have a function for sharing utility functions by appending to the injected function. Where as using model.factories will return the function with a value, not an empty function.
